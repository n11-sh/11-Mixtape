Planned Architectures

The architectures described here are planned targets, not guaranteed support. Compatibility will be introduced gradually, if technically viable.

Why Architecture Matters

Choosing supported CPU architectures is not a cosmetic decision. Each architecture affects kernel design, security assumptions, memory models, and isolation strategies. 11 Mixtape prioritizes architectures that allow strong control, clear documentation, and realistic long-term maintenance.

x86_64

x86_64 is planned as a primary target due to its maturity, extensive documentation, and widespread availability on desktops, laptops, and servers.

Supporting x86_64 allows 11 Mixtape to run on common hardware, test advanced memory protection techniques, and experiment with security-oriented kernel features without severe hardware limitations.

ARM64 (AArch64)

ARM64 is planned as a parallel target architecture due to its growing relevance in modern computing, including laptops, mobile devices, and low-power systems.

Its simpler design, strong privilege separation, and energy efficiency make it an interesting platform for a security-focused system. ARM64 also opens the possibility of running 11 Mixtape on future devices beyond traditional PCs.

Future Architectures

Support for additional architectures is not currently planned. Expanding compatibility increases complexity, maintenance burden, and potential security risk.

New architectures will only be considered if they align with the project's security goals and do not compromise architectural clarity.
