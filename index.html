11 Mixtape

⚠️ Reality check: 11 Mixtape is developed by a teenager, not a corporation, not a startup, and definitely not a team of kernel wizards. This project is experimental, unstable, and very much a work in progress. Bugs, broken ideas, and questionable decisions are part of the journey.

11 Mixtape is an experimental operating system kernel built around one stubborn idea: security should not be an afterthought. Instead of piling features first and fixing problems later, this project starts by asking “what if we just… didn’t trust anything by default?”

The result is a system that values isolation, restriction, and clarity over comfort, convenience, or mass appeal.

This is not a Linux replacement. It is not trying to beat existing systems, win benchmarks, or run every application under the sun. 11 Mixtape exists to learn, experiment, break assumptions, and sometimes hit a wall just to understand why that wall exists.

Progress is intentionally slow. Ideas are rewritten, discarded, and rebuilt often. If something feels unfinished, that’s because it probably is.

While security is the core focus, future versions may allow limited daily use. Think basic browsing, media playback, a graphical environment, and aggressive desktop customization (yes, ricing is allowed).

Heavy gaming, large proprietary ecosystems, and performance-hungry software are not the goal. Steam support is unlikely. If something runs, it’s because it earned the right to.

11 Mixtape is a long-term project driven by curiosity, paranoia, and the desire to understand systems at a deeper level. It is built openly, slowly, and without pretending to be something it isn’t.
